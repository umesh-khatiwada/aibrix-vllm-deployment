Architecture Overview: tp-1p1d-mi300x

This StormService implements strict prefill-decode disaggregation for Llama-3.2-1B-Instruct across AMD MI300X GPUs. Traffic is managed by a stateless CPU router. Prompt processing is isolated to a single MI300X (TP=1), which ships the KV cache via Mooncake to a highly-available, gang-scheduled decode cluster (TP=2) spanning two physical nodes.

🎭 Role Configuration Summary
🚦 Router • 1 replica

Runs aibrix/sglang-router with --pd-disaggregation. Polls port 30000 via K8s service discovery to maintain active maps of worker pods.

⚡ Prefill • 1 GPU (MI300X)

SGLang --disaggregation-mode prefill. Handles prompt processing. Transfers cache out via Mooncake backend.

🔄 Decode • 2 GPUs (TP=2)

Requires podGroupSize: 2 for atomic node scheduling. Rank mapped dynamically to cluster hostname.

Routing Layer & Service Discovery

The router pod operates on CPU and orchestrates traffic. Because it requires active pod discovery in the `default` namespace, it binds to the `sglang-router-sa` service account. It connects to the StormService pods using strict replica-index label selectors.

Container Spec: aibrix/sglang-router:v0.1.7-patch.1
python3 -m sglang_router.launch_router \ --host 0.0.0.0 \ --pd-disaggregation \ --policy random \ --service-discovery \ --service-discovery-port 30000 \ --worker-startup-timeout-secs 900 \ --prefill-selector \ storm-service-name=$STORM_SERVICE_NAME \ role-name=prefill \ stormservice.orchestration.aibrix.ai/role-replica-index=0 \ --decode-selector \ storm-service-name=$STORM_SERVICE_NAME \ role-name=decode \ stormservice.orchestration.aibrix.ai/role-replica-index=0 \ --service-discovery-namespace default

Prefill Processing Layer (TP=1)

The prefill node reserves 1 MI300X GPU. Instead of token generation, it mathematically processes the prompt, establishes the KV Cache, and transmits it via Mooncake. Hardware optimizations are enforced via OS-level environment variables to maximize AMD GPU throughput.

SGLang Engine Launch Params
python3 -m sglang.launch_server \ --model-path /models/Llama-3.2-1B-Instruct \ --served-model-name Llama-3.2-1B-Instruct \ --host 0.0.0.0 --port 30000 \ --disaggregation-mode prefill \ --disaggregation-transfer-backend mooncake \ --dtype bfloat16 \ --tp-size 1 \ --nnodes 1 \ --mem-fraction-static 0.8

Distributed Decode Coordination (TP=2)

This phase is memory-bandwidth bound and spread across 2 physical nodes. Because Kubernetes lacks native gang-scheduling, Aibrix manages podGroupSize: 2. To sync the distributed process over Ethernet without external dependencies like netcat, Rank-1 uses a pure Bash /dev/tcp polling loop to wait for Rank-0.

Dynamic Rank Derivation & Synchronization Script
# Derive Node Rank: Extracts index from <stormservice>-roleset-<id>-decode-<hash>-0-<RANK> NODE_RANK=$(echo $HOSTNAME | awk -F'-' '{print $NF}') # Derive Master DNS: Replaces trailing rank digit with 0 MASTER_HOSTNAME=$(echo $HOSTNAME | sed 's/-[0-9]*$/-0/') MASTER_DNS="${MASTER_HOSTNAME}.${STORM_SERVICE_NAME}.default.svc.cluster.local" MASTER_PORT=29500 # Rank-1 waits for Rank-0 (Master) to open port 29500 if [ "$NODE_RANK" != "0" ]; then until timeout 3 bash -c "echo > /dev/tcp/${MASTER_DNS}/${MASTER_PORT}" 2>/dev/null; do sleep 3 done fi # Launch Distributed Engine python3 -m sglang.launch_server \ --disaggregation-mode decode \ --tp-size 2 --nnodes 2 --node-rank $NODE_RANK \ --dist-init-addr ${MASTER_DNS}:${MASTER_PORT}
🏎️ Decode-Specific Configurations

CRD Architectural Skeleton

Aibrix uses the `StormService` Custom Resource to package the entire inference microservice into a single deployable object. By setting `stateful: true`, it guarantees stable network identities needed for multi-node Tensor Parallelism.

YAML Skeleton Outline orchestration.aibrix.ai/v1alpha1
apiVersion: orchestration.aibrix.ai/v1alpha1 kind: StormService metadata: name: tp-1p1d-mi300x spec: replicas: 1 updateStrategy: { type: InPlaceUpdate } stateful: true roles: # ── Router (CPU Only) ─────────────────────── - name: routing replicas: 1 stateful: true # ── Prefill (TP=1, GPU=1) ─────────────────── - name: prefill replicas: 1 stateful: true # ── Decode (TP=2, GPU=2) ──────────────────── - name: decode replicas: 1 podGroupSize: 2 # Gang-schedule across 2 nodes stateful: true # Required for rank/DNS derivation
